The superModule Keyword

To extend a controller safely, create a file with the same name in a higher-priority cartridge and use server.extend(module.superModule). This initiates the middleware inheritance chain, preserving the base logic while appending your custom functionality.

3. System Quotas & Platform Limits

Salesforce enforces hard platform limits (Quotas) to ensure multi-tenant stability. Exceeding these limits results in immediate fatal exceptions. Developers must design scripts to operate well within these constraints.

Script.MaxCallStackDepth Strict: 50

Deep recursion is strictly forbidden. The platform terminates any script execution thread that exceeds 50 nested function calls to prevent infinite loops.

Object.MaxCustomAttributes Schema Limit

Limits the number of custom attributes you can define on a single System Object type in Business Manager to prevent database schema bloat.

api.MaxObjectCount Memory Limit

Restricts the total number of persistent objects (like Products or Orders) that can be instantiated into memory during a single HTTP request.

4. Performance Optimization: Iterators vs Arrays

A common cause of Script.MaxMemory quota violations is using .toArray() or .asList() on massive database result sets. Visualized below is the memory impact of loading items into an Array versus streaming them directly from the database using a SeekableIterator.

✗ The Array Approach

Loading 50,000 orders into a List forces the JVM to allocate heap space for every single object simultaneously, guaranteeing a fatal memory crash.

✓ The Iterator Approach

A SeekableIterator streams records one-by-one. Combined with periodic transaction commits and clearing object references, memory usage remains flat.
