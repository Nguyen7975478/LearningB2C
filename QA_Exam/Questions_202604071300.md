**Summary:** This study guide focuses on the "Application Development" and "B2C Commerce Architecture" domains. It addresses common points of confusion regarding the SFRA Model-View-Controller pattern, efficient data processing to avoid platform quota violations, and the configuration of site-wide settings such as taxation and catalog visibility.

**Key Concepts:**

-   **SFRA Model Layer:** Unlike a database schema, an SFRA Model is a wrapper around a raw system object (like a Product or Basket). Its job is to "clean" the data—formatting currency, translating strings, or stripping complex logic—before passing a simplified JSON object to the ISML template.
-   **Platform Quotas (Memory & Script):** The `Script.MaxMemory` quota is often hit when processing large collections (like orders or products). The best practice is to use **Iterators** rather than converting collections to arrays, and to manually nullify or clear object references inside loops to allow for garbage collection.
-   **Taxation vs. Currency:** Tax settings are governed by the **Taxation Type** (Net vs. Gross), which is site-specific. This is distinct from the **Default Currency**, which only determines the monetary unit displayed.
-   **Cache Busting for Static Assets:** Always use `URLUtils.staticURL()` for CSS and JS. This ensures that the platform appends a version string to the URL, forcing the browser to download the new file immediately after a code deployment.
-   **OCAPI Security:** While `x-dw-client-id` identifies the app, **OAuth JWT tokens** are required in the `Authorization` header for any request involving sensitive data or customer-specific actions.

**Vocabulary List:**

-   **ismodule:** An ISML tag used to declare a custom, reusable UI component that accepts attributes.
-   **WebDAV:** The standard protocol used by IDEs (via Prophet Debugger) to upload code cartridges to a sandbox.
-   **Code Profiler:** A Business Manager tool used to analyze script execution time and memory consumption in real-time.
-   **Transaction.wrap():** A mandatory wrapper for any script logic that modifies persistent database objects; failing to use it results in a persistence exception.
-   **Information Tool:** A component of the Storefront Toolkit that visually displays the template inclusion stack and cartridge overrides.
