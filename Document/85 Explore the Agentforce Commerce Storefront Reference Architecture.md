### What’s SFRA?
- SFRA is more than just a code base: it provides a **blueprint for site design**. ([trailhead.salesforce.com](https://trailhead.salesforce.com/content/learn/modules/cc-digital-for-developers/digital-ref-architecture))
- Using a reference architecture is about **speedy innovation**:
    - build sites faster with a full library of core site capabilities built with industry best practices ([trailhead.salesforce.com](https://trailhead.salesforce.com/content/learn/modules/cc-digital-for-developers/digital-ref-architecture))
- The unit calls out out-of-the-box features, including:
    - cart, checkout, homepage, product detail pages, and more ([trailhead.salesforce.com](https://trailhead.salesforce.com/content/learn/modules/cc-digital-for-developers/digital-ref-architecture))
- It also highlights fast adoption of technologies (example: Apple Pay) to improve shopper experience. ([trailhead.salesforce.com](https://trailhead.salesforce.com/content/learn/modules/cc-digital-for-developers/digital-ref-architecture))
- SFRA customization model:
    - easy to build/maintain/update storefront
    - sits outside the platform API layer
    - uses conventional **MVC architecture** (tied back to the prior “development environment” unit). ([trailhead.salesforce.com](https://trailhead.salesforce.com/content/learn/modules/cc-digital-for-developers/digital-ref-architecture))
- Developers can customize/extend SFRA:
    - bring in new code
    - integrate with third-party providers
    - iterate on site designs
- Outcome: merchants own their storefront and underlying code base. ([trailhead.salesforce.com](https://trailhead.salesforce.com/content/learn/modules/cc-digital-for-developers/digital-ref-architecture))
