## What qualifiers are (refresher)
Qualifier types (unit list):
- **Coupons**: multiple or single-use codes shoppers use for discounts.
- **Customer groups**: lists of specific customers or customers who meet criteria (for example geographic location).
- **Source codes**: a code stored in a cookie when shoppers click an affiliate/newsletter link; can automatically trigger campaign experiences.

Qualifiers can be combined per promotion experience later inside a campaign (configured per experience).

## Coupons
Key points:
- Coupons can be created in B2C Commerce or imported from an external system.
- Coupons can exist outside campaigns/promotions.
- If you configure a coupon as a promotion qualifier, then when the promotion expires, the coupon expires too.
- You can assign a coupon to one or more promotions; a promotion can be associated with multiple coupons.
- If you associate a coupon with a campaign, all promotions in that campaign automatically inherit the coupon; you can also disinherit a coupon within a promotion.

### Coupon types (3)
1) **Merchant-defined single code**
- One fixed coupon code per coupon; can be used by multiple shoppers.
- Maintained by the merchant; add/delete in Business Manager.

2) **Merchant-defined multiple codes**
- List of codes per coupon (typically imported from external system); each code can be used by multiple shoppers.
- Maintained by the merchant; add/delete in Business Manager.

3) **System-generated**
- Codes generated by B2C Commerce; each code can be used by multiple shoppers.
- Issued via export or scripting API; format managed in Business Manager.

Redemption limits note:
- The unit references redemption limits (for example limiting how many times each shopper can use the coupon in a transaction).

### Create a coupon (merchant-defined single code) — steps
1) Merchant Tools → Online Marketing → **Coupons**
2) **New**
    - ID: `spring-code`
    - Case-sensitive: No
    - Type: Single Code
    - Coupon: `SPRINGSPORTS`
3) Save
4) General tab:
    - Description: `Spring sports coupon`
    - Enable
    - Redemption limits per coupon code: leave unlimited
    - Redemptions per customer: `1`
    - Redemptions per time period: `10 days, 1 redemption`
5) Save

Best practice note in unit:
- Test campaigns/promotions on a **staging instance** (B2C Commerce isn’t available in Trailhead Playground).

## Customer groups
- Customer groups let you show content slots, promotions, and sorting rules to specific customers.

### Customer group types (3)
1) **System**
- Everyone, Registered, Unregistered
- Pre-existing and cannot be changed.

2) **Static**
- Add shoppers manually or upload from a CRM system.

3) **Dynamic**
- Membership rule based on customer attributes (example: “birthday in 30 days”).
- Unit example use case: “lapsed shoppers” who haven’t visited for a defined time.

### Create customer groups — steps
Create Static group (loyalty program):
1) Merchant Tools → Customers → **Customer Groups**
2) New
3) ID: `loyalty-shoppers`
4) Type: Static
5) Save
6) Members tab → Assign → select customers → Assign → Save

Create Dynamic group (lapsed shoppers):
1) Merchant Tools → Customers → Customer Groups → New
2) ID: `lapsed-shoppers`
3) Type: Dynamic → Save
4) General: description “Shoppers that haven't visited the site in one year.”
5) Membership Rule:
    - Include Customers with → Select Attribute
    - Attribute: Visits (365)
    - Operator: is equal
    - Value: 0
6) Save

## Source codes
- Source codes can trigger different actions behind the scenes, including:
    - Direct customers to landing pages / featured PDPs / category lists / URLs
    - Redirect them to other sites
    - Activate price books
    - Enable promotions
- Source code as qualifier example: shopper clicks newsletter email → source code stored in cookie → qualifies for free shipping over $100.

### Source code groups and patterns
- You can create a source code group that contains literal codes or patterns that match multiple codes.
- Pattern example: `SPORT[1..10]`:
    - `SPORT1` matches
    - `sport1` does not match (case-sensitive behavior demonstrated)
- You can map multiple codes to the same rules (price books, promotions, redirects) while tracking performance per source code.
- You can assign one or more source codes to a campaign so all promotions in the campaign use those source codes.

### Create source codes — steps
1) Merchant Tools → Online Marketing → **Source Code Groups**
2) New
3) ID: `source-code-spring`
4) Enter description + start/end dates (must fall within campaign schedule)
5) Cookie duration (days): `30`
6) Enable → Apply
7) Codes tab:
    - Source code specification: `SPORT[1..10]`
    - Add
8) Test:
    - `sport1` → Test Match → does not match
    - `SPORT1` → Test Match → matches

## Qualifiers là gì (nhắc lại)
3 qualifier types trong unit:
- **Coupons**: codes single-use hoặc multiple-use để nhận discount.
- **Customer groups**: nhóm khách hàng cụ thể hoặc nhóm theo criteria (ví dụ location).
- **Source codes**: code lưu trong cookie khi shopper click link affiliate/newsletter; có thể tự động trigger experiences.

Có thể combine qualifiers cho từng promotion experience bên trong campaign (cấu hình sau).

## Coupons
Điểm chính:
- Coupon tạo trong B2C Commerce hoặc import từ external system.
- Coupon có thể tồn tại ngoài campaign/promotion.
- Nếu coupon là promotion qualifier, promotion expire thì coupon expire.
- 1 coupon gán cho nhiều promotions; 1 promotion có thể gán nhiều coupons.
- Coupon gán vào campaign → tất cả promotions trong campaign inherit coupon; có thể disinherit coupon trong từng promotion.

### 3 coupon types
1) **Merchant-defined single code**
- 1 code cố định; nhiều shoppers dùng được.
- Merchant maintain; add/delete trong Business Manager.

2) **Merchant-defined multiple codes**
- Danh sách codes (thường import); mỗi code nhiều shoppers dùng được.
- Merchant maintain; add/delete trong Business Manager.

3) **System-generated**
- Code do B2C Commerce generate; nhiều shoppers dùng được.
- Issued qua export hoặc scripting API; format quản lý trong Business Manager.

Redemption limits:
- Unit có nhắc redemption limits để giới hạn số lần dùng.

3) Save
4) General tab:
    - Description `Spring sports coupon`
    - Enable
    - Redemption limits per coupon code: unlimited
    - Redemptions per customer: `1`
    - Redemptions per time period: `10 days, 1 redemption`
5) Save

## Customer groups
- Dùng để show content slots, promotions, sorting rules cho nhóm customers cụ thể.

### 3 customer group types
1) **System**: Everyone, Registered, Unregistered (có sẵn, không đổi).
2) **Static**: add customers thủ công hoặc upload từ CRM.
3) **Dynamic**: rule dựa trên customer attributes (ví dụ birthday); ví dụ use case “lapsed shoppers”.

### Tạo customer groups — steps
Static loyalty:
- Customers → Customer Groups → New → ID `loyalty-shoppers` → Type Static → Save
- Members tab → Assign → chọn customers → Assign → Save

Dynamic lapsed:
- Customers → Customer Groups → New → ID `lapsed-shoppers` → Type Dynamic → Save
- Description “haven't visited the site in one year.”
- Membership Rule: Visits(365) is equal 0 → Save

## Source codes
- Có thể:
    - direct đến landing pages/featured PDP/category lists/URLs
    - redirect sang site khác
    - activate price books
    - enable promotions
- Ví dụ: click newsletter email → source code vào cookie → free shipping > $100.

### Source code groups/patterns
- Source code group chứa literal codes hoặc pattern match nhiều codes.
- Pattern `SPORT[1..10]`: `SPORT1` match, `sport1` không match.
- Map nhiều codes tới cùng rules (price books/promotions/redirects) nhưng vẫn track per code.
- Gán source codes vào campaign để promotions trong campaign dùng source codes đó.

### Tạo source codes — steps
- Online Marketing → Source Code Groups → New
- ID `source-code-spring`, dates trong campaign schedule, cookie duration 30, enable
- Codes tab: `SPORT[1..10]` → Add → Test Match với `sport1` và `SPORT1`
